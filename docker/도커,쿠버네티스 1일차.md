PC 환경에 따라 docker가 설치가 안되는 경우 
1) virutal-box나 vmware을 통해 가상 os로 linux를 설치하고 그 위에 docker를 설치하면 된다. (추천)
2) docker tool box를 설치해 보자.(p.30 Docker Toolbox)

* docker file의 경우  대문자 D로 표현 해줘야한다.

* dockerfile 기본형태

```dockerfile
FROM ubuntu:16.04

COPY helloworld /usr/local/bin
RUN chmod+x /usr/local/bin/helloworld

CMD["helloworld"]

## 해석
1) 인스트럭션 : FROM / COPY / RUN / CMD 등등
2) ubuntu:16.04 : ubuntu 이미지를 사용할 것이며 해당 이미지의 버전은 16.04이다.
3) 다른 인스트럭션과 다르게 CMD는 모든 Dockerfile에 단 하나만 존재한다.
```

* run 명령어

``` dockerfile
$ docker container run helloworld:latest
Hello, World!

## 해석
run 명령어의 의미 : 이미지를 Create 하고 Start 한다. 따라서 Stop 명령어를 통해 docker를 멈춘후 재가동하고 싶으면 start 명령어만 사용해도 재가동이 된다.
```

### 02) 도커를 사용하는 의의

> 01도커의 기초 p.11-18

* IaC (Infrastructure as code) : 프로그래밍형 인프라라고 불린다. 인프라 구성을 마치 소프트웨어를 프로그래밍하는 것처럼 처리하는 방식을 가리킨다. 이는 클라우트 컴퓨팅의 기본적인 부분으로 DevOps에 반드시 필요한 요소이다. 

* 멱등성 (idempotent) : 수학이나 전산학에서 연산의 한 성질을 나타내는 것으로, 연산을 여러 번 적용하더라도 결과가 달라지지 않는 성질을 의미한다.
* 미들웨어 : 응용 소프트웨어가 운영 체제로부터 제공받는 서비스 이외에 추가적으로 이용할 수 있는 서비스를 제공하는 컴퓨터 소프트웨어이다.
* 가상화 방식
  1) 기존 가상화 방식 : OS를 가상화. VMware,Virtualbox 등, Host OS위에 Guest OS 전체를 가상화하는 방식으로 **무겁고 느리다**.
  2) CPU의 가상화 기술 이용 방식 :  Kernel-based Virtual Machine. 전체 OS를 가상화하지 않으므로 Host형식에 비해 속도가 항상되었다. OPenStack, AWS 등의 클라우드 서비스 등. **추가적인 OS**는 여전히 필요하다.
  3) 프로세스 격리 : 리눅스 컨테이너. CPU나 메모리는 프로세스에 필요한 만큼만 추가로 사용한다.성능 손실 거의 없다

* Docker Image

  1) 컨테이너 실행에 필요한 파일과 설정 값 등을 포함한다 : 상태값이 아니다. 변할 수 없다(immutable).
  2) 이미지의 실체화 : Container 작업

* 이미지를 만드는 방법
  1) 처음부터 통짜로 하나  만든다
  2) 남이 만든 이미지를 가져와 해당 상태에서 부터 시작한다.

* `pull`명령어 

```dockerfile
docker pull 이미지

## 해석
docker hub에 있던 이미지를 local로 가지고 오기 위한 명령어. 해당 이미지를 `run`명령어를 통해 작동하게 된다.
```
